Enforcement & Governance - How The System Polices Itself
The Structural Enforcement Pattern
When a behavioral rule fails across two or more sessions, it graduates from "remember to do X" into a structural precondition built into an artifact - the skill cannot complete until X is verified. 10+ documented instances. This is the licensing thesis: the discipline lives in the system, not in the practitioner's memory. It is what lets the methodology run without the founder in the room.
25 load-bearing protocols, walked by session-init at the start of every session. Click any group to expand.
Session Gates
Fire before work begins
session-init - first tool call of every session; walks the full Protocol Index and produces a session brief.

pre-flight - Step 0 of every spec-governed skill. Loads specs, prior decisions, open loops, and runs the CCI Currency Check. Enforcement: no brief, no execution.
Delivery Gates
Fire before anything ships
qc-gate - 13-step QC sequence with a saved log. Enforcement: no QC log, no ship.

Ship Gate - standing item in every client status file; stays OPEN until both the QC log and the Register Pre-Delivery QC exist.

voice-vetting - structural Voice Fidelity check for ad-hoc voice output.
Voice Integrity
Protects authentic voice
Voice Fidelity + Mode Architecture - four modes (Write TO / AS / FOR Them, AS Their Company) identified before any voice output; sources read in full first.

CCI Currency Check - cascade audit confirming every transcript and feedback note has flowed into the canonical CCI before downstream skills run.
System Integrity
Keeps the system coherent
Cross-File Verification - any edit touching 5+ files cannot be declared complete until a grep returns clean.

Information Integrity - materiality scan every session + supersession registry when something true stops being true.

Handoff Protocol - completeness gate for methodology bundles shipping to licensing partners and certified facilitators.
Firing order: session-init → pre-flight (per skill) → skill executes → qc-gate + Register QC → Ship Gate → storyeq-debrief. Governance wraps delivery; it is not a static checklist.

How Connectors Map to Work
Superhuman / Gmail
Email intake + CRM sync
Client communications, transcript delivery, discovery call confirmations, proposal sends. Email-to-CRM sync fires when emails relate to CRM contacts. Email action items sync to TASKS. Draft-only - never sends without approval.
Google Calendar
Session scheduling
Client session scheduling, discovery calls, engagement timing. Read + create events. Confirm before modifying existing events. New prospect detection + calendar-to-TASKS sync via upcoming calls.
Notion
CRM + publishing
CRM database for pipeline management. PSR publishing via notion-psr-publisher. SOP knowledgebase. Thinking Ledger + Open Loops. Read freely, confirm before writes.
Otter.ai
Transcript capture + sync
Session and meeting transcripts. Action items and notable moments sync into the Project Log, CRM records, and TASKS. Transcripts route automatically to the correct client folder after a completed meeting.
Figma
Design system
Visual artifacts, brand guidelines reference, component systems. Used for branded visuals and design system consistency.
Google Drive

Canva
